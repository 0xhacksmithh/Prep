# 1. What is a Database?

A database is an organized collection of structured or semi-structured data stored electronically, designed for efficient storage, retrieval, and manipulation.

### Why databases are needed

Without databases:

- Data would be stored in files → slow search

- No concurrency control

- No security or backup

### Example

Imagine a **users table:**

| id  | name  | email                           |
| --- | ----- | ------------------------------- |
| 1   | Alice | [a@mail.com](mailto:a@mail.com) |
| 2   | Bob   | [b@mail.com](mailto:b@mail.com) |

Fetching Bob’s email:

```
SELECT email FROM users WHERE name = 'Bob';
```

### Real-world use

- Banking systems

- E-commerce (orders, users, payments)

- Social media platforms

# 2. What is DBMS?

### DBMS (Database Management System) is software that:

- Stores data

- Manages access

- Ensures security

- Handles concurrency

- Performs backup & recovery

### Popular DBMS examples

- MySQL

- PostgreSQL

- MongoDB

- Oracle

### Responsibilities

- Query processing

- Transaction management

- Data integrity

- Authorization

### Example

**Without DBMS:**

- You manually edit files

**With DBMS:**

```
INSERT INTO users VALUES (3, 'Charlie', 'c@mail.com');
```

# 3. Difference between DBMS and RDBMS

### DBMS

- Stores data in files

- No relationships

- No normalization

### RDBMS

- Stores data in tables

- Enforces relations

- Uses keys and constraints

### Comparison Table

| Feature        | DBMS        | RDBMS  |
| -------------- | ----------- | ------ |
| Data structure | Files       | Tables |
| Relations      | ❌          | ✅     |
| Keys           | ❌          | ✅     |
| ACID           | ❌          | ✅     |
| Example        | File system | MySQL  |

### Example

RDBMS relationship:

```
Orders.user_id → Users.id
```

# 4. What is a Table?

A table is a structured format to store data using:

**Rows** → records

**Columns** → attributes

### Example

`employees` table:

| emp_id | name  | salary |
| ------ | ----- | ------ |
| 1      | Rahul | 50000  |
| 2      | Anita | 60000  |

### Insert example

```
INSERT INTO employees VALUES (3, 'Suman', 70000);
```

### Key point

Each row represents one entity instance.

# 5. What is a Primary Key?

A primary key uniquely identifies each row in a table.

### Properties

- Unique

- Not NULL

- One per table

### Example

```
CREATE TABLE users (
  id INT PRIMARY KEY,
  email VARCHAR(100)
);
```

### Invalid case ❌

```
INSERT INTO users VALUES (1, NULL); -- Not allowed
```

### Real-world analogy

Like Aadhaar number — unique for every person.

# 6. What is a Foreign Key?

A foreign key creates a relationship between two tables by referencing the primary key of another table.

### Example

```
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**Meaning**

- Each order must belong to a valid user

- Prevents orphan records

### Example insert

```
INSERT INTO orders VALUES (101, 1); -- Valid
INSERT INTO orders VALUES (102, 99); -- ❌ Invalid user
```

# 7. What is Normalization?

**Normalization** is the process of organizing data to:

- Remove redundancy

- Avoid anomalies

- Improve integrity

### Problem (Unnormalized)

| order_id | customer | products       |
| -------- | -------- | -------------- |
| 1        | Alice    | Phone, Charger |

### Normalized

**customers**

| id  | name  |
| --- | ----- |
| 1   | Alice |

**orders**

| id  | customer_id |
| --- | ----------- |
| 1   | 1           |

**order_items**

| order_id | product |
| -------- | ------- |

### Benefit

- Easier updates

- Less duplication

# 8. What are Normal Forms?

### 1NF (First Normal Form)

- Atomic values

- No repeating groups

❌ `skills = "JS, Node"`
✅ Separate rows

### 2NF

- Must be in 1NF

- No partial dependency

❌ `order_id → product_name`
✅ Depends on full key

### 3NF

- No transitive dependency

❌ `user_id → city → pincode`
✅ Separate table

### BCNF

Stricter version of 3NF ensuring every determinant is a candidate key.

# 9. What is Denormalization?

Denormalization intentionally introduces redundancy to improve read performance.

### Example

**Normalized:**

```
Orders → Users → Address
```

**Denormalized:**

```
Orders (user_name, address)
```

### Tradeoff

| Advantage    | Disadvantage      |
| ------------ | ----------------- |
| Faster reads | Data duplication  |
| Fewer joins  | Update complexity |

### Used in

- Analytics systems

- High-read APIs

# 10. What is SQL?

**SQL** (Structured Query Language) is used to interact with relational databases.

### SQL operations

- Create tables

- Insert data

- Fetch records

- Update & delete data

### Example queries

```
SELECT * FROM users;
UPDATE users SET email='x@mail.com' WHERE id=1;
DELETE FROM users WHERE id=2;
```

### SQL is:

- Declarative (what, not how)

- Standardized

- Supported by all RDBMS
