# 11. Types of SQL Commands

SQL commands are grouped based on what they do.

### 1️⃣ DDL – Data Definition Language

**Used to define database structure.**

```
CREATE TABLE users (
  id INT PRIMARY KEY,
  name VARCHAR(50)
);
```

**Commands:**

- CREATE

- ALTER

- DROP

- TRUNCATE

### 2️⃣ DML – Data Manipulation Language

**Used to modify data.**

```
INSERT INTO users VALUES (1, 'Alice');
UPDATE users SET name='Bob' WHERE id=1;
DELETE FROM users WHERE id=1;
```

### 3️⃣ DQL – Data Query Language

**Used to fetch data.**

```
SELECT * FROM users;
```

### 4️⃣ DCL – Data Control Language

**Controls permissions.**

```
GRANT SELECT ON users TO admin;
REVOKE INSERT ON users FROM guest;
```

### 5️⃣ TCL – Transaction Control Language

**Manages transactions.**

```
BEGIN;
COMMIT;
ROLLBACK;
```

# 12. What is a NULL Value?

**NULL** represents missing or unknown data.

**Important points**

- NULL ≠ 0

- NULL ≠ empty string

### Example

```
INSERT INTO users (id, email) VALUES (1, NULL);
```

### Checking NULL

```
SELECT * FROM users WHERE email IS NULL;
```

### ❌ Wrong:

```
WHERE email = NULL;
```

# 13. Difference Between DELETE and TRUNCATE

| Feature  | DELETE        | TRUNCATE      |
| -------- | ------------- | ------------- |
| Deletes  | Selected rows | All rows      |
| WHERE    | Allowed       | Not allowed   |
| Rollback | Yes           | No            |
| Speed    | Slower        | Faster        |
| Triggers | Fires         | Does not fire |

### Example

```
DELETE FROM users WHERE id=5;
TRUNCATE TABLE users;
```

# 14. What is a Constraint?

Constraints enforce rules on data to maintain integrity.

### Types of constraints

### NOT NULL

```
name VARCHAR(50) NOT NULL
```

### UNIQUE

```
email VARCHAR(100) UNIQUE
```

### PRIMARY KEY

```
id INT PRIMARY KEY
```

### FOREIGN KEY

```
FOREIGN KEY (user_id) REFERENCES users(id)
```

### CHECK

```
age INT CHECK (age >= 18)
```

# 15. What is Indexing?

Indexing improves data retrieval speed.

### Without index

- Full table scan

### With index

- Faster lookup using B-Tree or Hash

### Example

```
CREATE INDEX idx_email ON users(email);
```

### Trade-off

Pros Cons
| Pros | Cons |
| ------------- | -------------------- |
| Faster SELECT | Slower INSERT/UPDATE |
| Better search | Extra storage |

# 16. What is a Clustered Index?

A clustered index determines the physical order of data.

### Key points

- Only one per table

- Data stored in sorted order

### Example

```
PRIMARY KEY (id)
```

### Visualization

```
Table data → sorted by id
```

# 17. What is a Non-Clustered Index?

A non-clustered index stores pointers to table rows.

### Key points

- Multiple allowed

- Does not change physical order

### Example

```
CREATE INDEX idx_name ON users(name);
```

**Difference**
| Clustered | Non-Clustered |
| ----------- | ------------------ |
| One only | Many |
| Data sorted | Separate structure |

# 18. What is ACID?

ACID ensures reliable transactions.

### A – Atomicity

All or nothing.

```
Transfer ₹100 → succeed or rollback
```

### C – Consistency

DB moves from one valid state to another.

### I – Isolation

Transactions don’t interfere.

### D – Durability

Committed data survives crashes.

# 19. What is a Transaction?

A transaction is a sequence of SQL operations executed as a single unit.

### Example

```
BEGIN;
UPDATE account SET balance = balance - 500 WHERE id=1;
UPDATE account SET balance = balance + 500 WHERE id=2;
COMMIT;
```

**If error occurs**

```
ROLLBACK;
```

# 20. What is a Join?

A JOIN combines rows from multiple tables using a related column.

### Example tables

**users**
| id | name |
| -- | ----- |
| 1 | Alice |

**Join query**

```
SELECT users.name, orders.id
FROM users
JOIN orders ON users.id = orders.user_id;
```

**Result**
| name | order_id |
| ----- | -------- |
| Alice | 101 |
