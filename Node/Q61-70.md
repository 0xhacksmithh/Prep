# 61. How Do You Secure a Node.js Application?

### Explanation

Security in Node.js is **multi-layered**.

### Key Practices

- Input validation

- Authentication & authorization

- Rate limiting

- Secure headers

- Dependency scanning

### Example

```
app.use(helmet());
app.use(rateLimiter);
app.use(cors());
```

### Interview Tip

Security is not one feature—it’s a process.

# 62. How Do You Validate Input in Node.js?

### Explanation

Input validation prevents:

- SQL/NoSQL injection

- XSS

- Invalid data

### Example (Zod)

```
import { z } from "zod";

const schema = z.object({
  email: z.string().email(),
  password: z.string().min(8)
});

schema.parse(req.body);
```

### Interview Tip

Always validate at API boundaries.

# 63. What is bcrypt and Why Is It Used?

### Explanation

`bcrypt` is a password hashing algorithm.

Why bcrypt?

- Salted hashing

- Slow by design (brute-force resistant)

### Example

```
import bcrypt from "bcrypt";

const hash = await bcrypt.hash(password, 10);
const match = await bcrypt.compare(password, hash);
```

### Interview Tip

Never store plain passwords.

# 64. Hashing vs Encryption

| Hashing   | Encryption     |
| --------- | -------------- |
| One-way   | Two-way        |
| Passwords | Sensitive data |
| No key    | Requires key   |

### Interview Tip

- Passwords → Hash
- Secrets → Encrypt

# 66. How Do You Manage Secrets in Node.js?

### Explanation

Secrets should never be in source code.

### Options

- .env files

- Vault (AWS Secrets Manager, HashiCorp Vault)

- CI/CD secrets

### Example

# 68. What is MongoDB?

### Explanation

MongoDB is a NoSQL, document-based database.

### Features:

- JSON-like documents

- Flexible schema

- Horizontal scaling

### Example

```
db.users.insertOne({ name: "John", age: 30 });
```

# 69. What is Mongoose?

### Explanation

Mongoose is an `ODM (Object Data Modeling)` library for MongoDB.

### Provides:

- Schema validation

- Middleware

- Query helpers

### Example

```
const UserSchema = new mongoose.Schema({
  email: String,
  password: String
});
```

# 70. SQL vs NoSQL

| SQL          | NoSQL                |
| ------------ | -------------------- |
| Fixed schema | Flexible schema      |
| ACID         | Eventual consistency |
| Joins        | Embedded docs        |

### Interview Tip

Choose DB based on use case, not hype.
