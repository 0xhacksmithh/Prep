# ðŸŸ¢ 22. What is \_\_dirname?

### Explanation

**\_\_dirname** is a global variable that returns the absolute path of the current directory where the file is located.

**It helps in:**

- Resolving file paths safely

- Avoiding path issues across OS

### Example

```
console.log(__dirname);

const path = require("path");

const filePath = path.join(__dirname, "data", "file.txt");
```

### Interview Tip

Always use `__dirname` instead of relative paths in backend apps.

# ðŸŸ¢ 24. What is the fs module?

### Explanation

The `fs` (File System) module allows Node.js to read, write, update, and delete files.

**It supports:**

- Async (non-blocking)

- Sync (blocking)

### Example (Async â€“ Recommended)

```
const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
  if (err) throw err;
  console.log(data);
});
```

### Example (Sync â€“ Avoid in prod)

```
const data = fs.readFileSync("data.txt", "utf8");
```

### Interview Tip

Always prefer async file operations in production.

# ðŸŸ¢ 26. What is the http module?

### Explanation

The http module allows you to create a web server without using Express.

### Example

```
const http = require("http");

const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Hello Node Server");
});

server.listen(3000);
```

### Interview Tip

Express is built on top of http.

# ðŸŸ¢ 28. What is Middleware?

### Explanation

Middleware functions sit between `request` and `response` and can:

- Modify request

- End request

- Pass control

### Example

```
app.use((req, res, next) => {
  console.log(req.method);
  next();
});
```

### Interview Tip

Order of middleware matters.

# ðŸŸ¢ 29. What is Express.js?

### Explanation

Express.js is a minimal, flexible web framework for Node.js that simplifies:

- Routing

- Middleware

- HTTP handling

### Example

```
import express from "express";
const app = express();

app.get("/", (req, res) => {
  res.send("Hello Express");
});

app.listen(3000);
```

# ðŸŸ¢ 30. Node.js vs Express.js

### Explanation

| Node.js        | Express.js   |
| -------------- | ------------ |
| Runtime        | Framework    |
| Low-level      | High-level   |
| Manual routing | Easy routing |

### Interview Tip

Node handles execution, Express handles structure.
