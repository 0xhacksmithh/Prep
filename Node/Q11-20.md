# üü¢ 11. What is require() in Node.js?

### Explanation

`require()` is a **CommonJS** function used to `import` modules into another file.

Node resolves modules in this order:

- Core modules (`fs, path`)

- Local files (`./utils.js`)

- `node_modules`

### Example

```
// math.js
function add(a, b) {
  return a + b;
}
module.exports = add;
```

```
// index.js
const add = require("./math");
console.log(add(2, 3)); // 5
```

### Interview Tip

`require()` is **synchronous**, meaning it blocks execution during loading.

# üü¢ 12. What is module.exports?

### Explanation

`module.exports` defines what a `module` exposes to other files.

Node wraps each file like:

```
(function(exports, require, module) {
// your code
});
```

### Example

```
module.exports = {
add: (a, b) => a + b,
sub: (a, b) => a - b
};
```

```
const math = require("./math");
math.add(5, 2);
```

### Interview Tip

`exports` is just a **reference** to `module.exports`.

# üü¢ 13. What is the Event Loop?

### Explanation

The **event loop** allows Node.js to execute non-blocking async operations on a single thread.

Instead of waiting, tasks are delegated and callbacks are executed later.

### Event Loop Phases (simplified)

- Timers

- I/O callbacks

- Poll

- Check

- Close callbacks

### Example

```
console.log("Start");

setTimeout(() => {
console.log("Timeout");
}, 0);

console.log("End");
```

### Output

```
Start
End
Timeout
```

### Interview Tip

Node is single-threaded, but not blocking.

# üü¢ 14. What are Callbacks?

### Explanation

A callback is a function passed as an argument and executed after an async operation finishes.

### Example

```
function fetchData(callback) {
setTimeout(() => {
callback("Data received");
}, 1000);
}

fetchData((data) => {
console.log(data);
});
```

### Problem

Callbacks can lead to callback hell.

# üü¢ 16. What are Promises?

### Explanation

A Promise represents a future value that can be:

- fulfilled

- rejected

- pending

### Example

```
const promise = new Promise((resolve, reject) => {
setTimeout(() => {
resolve("Success");
}, 1000);
});

promise.then(console.log).catch(console.error);
```

### Interview Tip

Promises flatten callback hell.

# üü¢ 18. What is async/await?

### Explanation

`async/await` is syntactic sugar over promises that makes async code look synchronous.

### Example

```
async function getData() {
const data = await fetchData();
console.log(data);
}
```

### Error Handling

```
try {
await riskyOperation();
} catch (err) {
console.error(err);
}
```

# üü¢ 19. What is the process object?

### Explanation

`process` gives access to **Node.js runtime info**.

Common uses:

- Environment

- Exit process

- Signals

### Example

```
console.log(process.pid);
console.log(process.platform);
```

# üü¢ 20. What is process.env?

### Explanation

`process.env` stores environment variables, commonly used for:

- Secrets

- Config

- Ports

### Example

```
const PORT = process.env.PORT || 3000;
```

Using `.env`:

```
PORT=5000
```

```
import "dotenv/config";
```

**Interview Tip**

Never hardcode secrets.
